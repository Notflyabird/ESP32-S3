### 1.4 核心代码 （[main/main.c](../main/main.c)）

```c
#include <stdio.h>
#include "freertos/FreeRTOS.h"
#include "freertos/task.h"
#include "esp_log.h"

static const char *TAG = "main";

void app_main(void)
{
    ESP_LOGI(TAG, "ESP32-S3 项目启动成功！");

    while (1) {
        printf("Hello from ESP32-S3!\n");
        vTaskDelay(pdMS_TO_TICKS(1000));   // 每秒打印一次
    }
}
```

## 三、编译方法

### 方式 1：VS Code 内编译（推荐）

1. **`Ctrl+Shift+P`** → `ESP-IDF: Build your project`
2. 或点击底部状态栏的 **`🔨 Build`** 按钮

### 方式 2：ESP-IDF CMD 终端

打开 **开始菜单 → ESP-IDF 5.5 CMD**，然后执行：

```batch
d:
cd \03_project\esp32\01_code\esp32-s3
idf.py build
```

### 方式 3：VS Code 内嵌终端（cmd）

```batch
C:\Espressif\python_env\idf5.5_py3.11_env\Scripts\python.exe ^
    C:\Espressif\frameworks\esp-idf-v5.5.4\tools\idf.py build
```

### 编译输出

```
build/
├── esp32_s3_test.bin          # 主应用固件 (~190KB)
├── bootloader/bootloader.bin  # 启动引导程序 (~20KB)
└── partition_table/           # 分区表
```

> ⚠️ **注意**：不要使用 Git Bash / MSYS2 环境编译，ESP-IDF 官方不支持 MinGW。

## 四、烧录方法

### 方式 1：VS Code 一键烧录

**`Ctrl+Shift+P`** → `ESP-IDF: Flash your project`  
或点击底部状态栏的 **`⚡ Flash`** 按钮

### 方式 2：ESP-IDF CMD 终端

```batch
idf.py -p COM3 -b 115200 flash
```

### 方式 3：烧录 + 串口监视

```batch
idf.py -p COM3 -b 115200 flash monitor
```

退出监视器按 `Ctrl + ]`。

### 硬件连接

| 引脚 | 说明 |
|---|---|
| **USB** | 连接电脑（供电 + 烧录 + 串口） |
| **COM3** | 默认串口端口（可在设备管理器确认） |
| **BOOT 键** | 按住 → 按 EN → 松开 BOOT 进入下载模式（IDF 通常自动处理） |

## 七、常见问题

### Q: 编译时报 `CMAKE_MAKE_PROGRAM is not set`

**原因**：在 Git Bash 中编译，Ninja 路径未正确传递。

**解决**：使用 ESP-IDF CMD 或 VS Code 插件编译。

### Q: 编译报 `app_check_size` 找不到 `.bin` 文件

**原因**：构建缓存不一致（之前编译被中途打断）。

**解决**：

```batch
idf.py fullclean
idf.py build
```

### Q: 串口监视看不到输出

**原因**：波特率不匹配 或 串口号错误。

**解决**：

```batch
idf.py -p COM正确端口 -b 115200 monitor
```
